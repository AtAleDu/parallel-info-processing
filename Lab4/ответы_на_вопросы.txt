1. OpenMP — это стандарт для параллельного программирования в системах с общей памятью. 
Он реализует модель общей памяти (shared memory), где все нити имеют доступ к общим переменным,
 и модель fork-join, при которой основной поток создаёт параллельные нити и после их завершения объединяется с ними.

2. OpenMP поддерживается в языках C, C++ и Fortran. В Visual Studio он состоит из поддержки компилятора,
 библиотеки выполнения OpenMP и директив #pragma omp для управления параллелизмом.

3. Количество нитей можно задать с помощью функции omp_set_num_threads(), 
переменной окружения OMP_NUM_THREADS, параметра num_threads в директиве #pragma omp parallel или автоматически системой.

4. Нити идентифицируются с помощью функции omp_get_thread_num(), которая возвращает номер нити.
 Это нужно для различения нитей и распределения работы. Например, каждая нить может вывести свой номер.
  Эти идентификаторы не совпадают с идентификаторами потоков операционной системы.

5. Порядок вывода сообщений нитями не фиксирован и может меняться при каждом запуске. 
Он определяется планировщиком операционной системы и скоростью выполнения нитей.

6. Частные переменные — это переменные, у которых каждая нить имеет свою копию. 
Они нужны для предотвращения конфликтов при одновременном доступе. Это не противоречит модели общей памяти, 
так как переменные специально делаются частными. 
Например, переменная счётчика цикла может быть частной.

7. В параллельной программе появляется параллельная область и область нити. 
Они задаются с помощью директивы #pragma omp parallel и других директив OpenMP.

8. Конфликт возникает, когда несколько нитей одновременно изменяют переменную rank. 
Это может привести к неправильному результату. Конфликт можно предотвратить, 
сделав переменную частной или используя директивы critical или atomic.

9. Результаты параллельного выполнения записываются в общие переменные, которые используются после завершения параллельной области.

10. Параметр reduction создаёт частную копию переменной для каждой нити и объединяет результаты после завершения работы. 
Частные переменные инициализируются начальным значением операции, например нулём для сложения.



Критическая секция — это участок программы, который может выполнять только один поток в один момент времени.



Почему номера потоков начинаются с 0?
Потому что нумерация потоков в OpenMP начинается с нуля.

Почему порядок строк может быть разный?
Потому что потоки выполняются параллельно, и невозможно заранее определить, какой поток выполнится раньше.

Зачем нужен reduction, если есть localSum?
Ответ:
Reduction используется для безопасного объединения результатов потоков в общей переменной totalSum без возникновения race condition.

reduction означает:
у каждого потока есть своя копия totalSum
потоки считают независимо
после завершения OpenMP складывает результаты
